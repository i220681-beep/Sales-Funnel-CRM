Data Structures, Made Visible

Sales Funnel CRM

Started as a terminal app for a Data Structures course — a Queue manages incoming leads, a Stack manages follow-ups, and classic sorting/searching algorithms organise the customer book. Rebuilt here as a live demo so you can watch the structures work, not just read about them.

Ibtisam Kayani & Ahsan Chohan · source on GitHub →

Lead Queue — FIFO

New leads join at the right. The oldest lead (left) is always processed first — nobody gets skipped.

Follow-up Stack — LIFO

Push a customer onto the stack to flag a follow-up. The most recent one is handled first, like a to-do list responding to what just happened.

Customer Book

Sort by deal potential, or search by name / ID. Every comparison the algorithm makes streams into the console below.
